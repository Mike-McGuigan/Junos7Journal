# Content Model

The project is moving toward a publishing model with one source of truth.

## Main idea

Content lives in `content/`.

Outputs live in:

- `site/`
- `collector/`
- `exports/`

The existing `website/`, `journal/`, `maps/` and `print/` folders can remain for now. Future releases will gradually migrate material into the cleaner structure.

## Why this matters

The same journal data should eventually generate:

- website pages
- printed book pages
- route maps
- statistics
- video indexes
- archive manifests
